Person endpoints

GetPerson
api/Person/completereturn all the "Person"s currently in the database, with all inforamtion
api/Person/complet/idReturn a specific person with all information
api/Person/contactinfo Returns all persons in the db's contact information
api/Person/contactinfo/idReturns a specific persons contact information
api/Person/city/zipcodeReturns all persons within that zipcode.
PUTPerson
api/Person/id Takes a person id and a person object and updates current person to equal the new/updated Person
POSTPerson
api/PersonTakes a Person json object and add it to database if it is valid

Hobby endpoints

GET
api/HobbyReturns all the hobbbys in the databse
api/Hobby/idReturns a specific hobby from the databse with the corresponding id.
api/Hobby/getHobbyPopulation/idReturns a amount of people a personList on Hobby
PUT
api/Hobby/idUpdates current Hobby with the corresponding id, to the new hobby passed in.
DELETE
api/Hobby/idDeletes the hobby with the corresponding id, and returns the object if the method was succesfull
POST
api/Hobby/createHobbyCreate a new Hobby given a JSON object (Using this request it should be possible to create a Hobby with name and Description)

Address endpoints

GET
api/AdressReturns all the Addresses in the database as a json object/array
api/Address/idReturns a specific date corresponding to the id, as a json object
POST
api/Address/createAddressCreate a new Address given a JSON object (Using this request it should be possible to create a Hobby with street, additional info additional a cityInfor object)
DELETE
api/Address/idDeletes the address corresponding to the id, if it exist
PUT
api/Address/idTakes a json object and updates the corresponding Address with the new values

CityInfo endpoints

Takes a json object and updates the corresponding CityInfo object with the new values
GET
api/CityInfo/zipCodeListReturns a json array of all citys and zipcodes in Denmark
api/CityInfo/{zipCode}Returns a json object with zipcode and corresponding city
POST
api/CityInfoTakes a CityInfo object and adds it to the database
DELETE
api/CityInfo/{zipCode}Deletes the CityInfo object corresponding to the id/zipcode, if it exist
PUT
api/CityInfo/{zipCode}
